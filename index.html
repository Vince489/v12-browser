The Technology

Understanding the VIRT:// Protocol

A custom browser protocol that transforms any GitHub repository into a browsable website

How It Works

1

Register Your Domain

Visit virt://register.at and claim your domain name (.vc, .vmc, .at, or .lit)

2

Link Your GitHub Repo

Point your domain to any public GitHub repository containing your website files

3

Instant Deployment

Your site is live immediately. Push updates to GitHub and they appear in real-time

4

Get Discovered

Your site is automatically indexed by lookin.at for the community to find

EXAMPLE REGISTRATION
Domain: my-awesome-project
TLD: .vc
GitHub URL: https://github.com/username/my-awesome-project
Branch: main

Result: virt://my-awesome-project.vc

Parallel Internet Mode

Host Anywhere.
Browse Everywhere.

V12 is more than a GitHub viewer. It's a decentralized routing layer for the modern pioneer. Point your virt:// domains to any IP, any port, or any local service.

  • Zero-Config Intranets: Build a private company or family network using the .vc TLD pointing to internal server IPs.

  • Localhost Aliasing: Stop typing IP addresses. Map virt://dev.at to your current build for seamless local testing.

  • Hardware Integration: Point a domain to a Raspberry Pi or IoT gateway for a clean, human-readable interface to your hardware.

PRO TIP: Use Zipidy to index your private IP sites by allowing the crawler access to your local network.
